设计模式

老羊肖恩2023/11/14

# 介绍

解决一些问题的最佳实践

设计模式是软件开发人员在软件开发过程中面临的一般问题的解决方案。这些解决方案是众多软件开发人员经过相当长的一段时间的试验和错误总结出来的。

# 七大原则

  • 开闭原则：对扩展开发，对修改关闭
  • 单一职责原则：对于类来说，一个类只负责一项职责
  • 依赖倒转原则：面向接口编程，依赖抽象，而不是具体实现，减少耦合
  • 接口隔离原则：用多个接口，细化接口，而不是使用单一的接口
  • 里氏替换原则：继承复用的基石，对开闭原则的补充
    • 子类可以扩展父类的功能，但不能修改父类原有的功能
    • 子类可以实现父类的抽象方法，但不能覆盖原有的父类方法
  • 合成复用原则：尽量使用合成/聚合的方式，而不是使用继承
  • 迪米特原则：最少知道原则，对于被依赖的类不管有多复杂，将其逻辑封装到内部，不向外部泄露信息
    • 降低类与类之间的耦合性

# 分类

  • 创建型：隐藏对象创建过程，通过逻辑方法进行对象的创建
  • 结构型：关注类和对象的组合关系
  • 行为型：关注对象之间的通信

# 工厂方法模式

定义一个接口或抽象工厂类，让它的实现类来创建具体的对象，不同的工厂实现类创建不同的对象

工厂方法模式生产的对象都是同一系列的对象

# 抽象工厂模式

提供了创建一系列相关或者相互依赖的对象的接口。类似公司下的产品，生产汽车，将汽车分为各个零部件进行生产：车架，车门，底盘等

# 建造者模式

将复杂的对象的创建和属性的复制分离，创建的过程和细节我们不需要知道，只需要通过构建者去操作即可

# 原型模式

用于创建重复的对象，保证创建对象的性能。原型模式是创建对象的最佳方式
